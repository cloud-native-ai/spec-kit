# Feature Detail: Claude Code Support

**Feature ID**: 021  
**Name**: Claude Code Support  
**Description**: Add Claude Code as a first-class assistant with custom commands and Claude Code-specific configuration assets.  
**Status**: Implemented  
**Created**: 2026-05-14  
**Last Updated**: 2026-05-14

## Overview

Claude Code Support enables Spec Kit users to create or refresh workspaces that are ready for Claude Code. The feature aligns assistant selection, generated project guidance, custom workflow command discovery, ignore configuration, documentation, and validation so Claude Code users can follow the same spec-driven workflow without manual setup.

## Latest Review

Generated implementation plan `009-claude-code-support/plan.md` plus Phase 0/1 design artifacts (`research.md`, `data-model.md`, `contracts/claude-code-support.openapi.yaml`, `quickstart.md`, and `feature-ref.md`), then decomposed the work into executable tasks in `009-claude-code-support/tasks.md`. The task breakdown confirms Feature 021 remains the sole primary feature, keeps Feature 020 Qoder Support as the closest implementation pattern, and sequences the Claude Code governance update as a foundational release blocker before user-story implementation can be considered complete.

## Key Changes

1. Track Claude Code as a first-class supported assistant in the feature registry.
2. Define requirements for Claude Code custom command surfaces that map to canonical Spec Kit workflows.
3. Define requirements for Claude Code-specific ignore configuration that protects local, generated, dependency, cache, and sensitive content.
4. Require refresh behavior that preserves existing assistant integrations and user customizations where safe.
5. Require consistency audits across setup guidance, generated assets, documentation, and release artifacts.
6. Added planning artifacts for assistant metadata, command coverage, ignore policy, refresh behavior, support-surface audits, validation scenarios, and feature references.
7. Recorded the governance update as a blocking implementation concern because the current constitution-approved assistant list must include Claude Code before release readiness.
8. Added executable task breakdown with 66 tasks across setup, foundational governance, three user-story phases, and polish/regression validation.
9. Prioritized US1 as the MVP for new-project Claude Code readiness, followed by existing-workspace refresh safety and release-wide consistency audits.

## Implementation Notes

- Treat Claude Code-specific files as compatibility assets derived from canonical Spec Kit instructions.
- Preserve the many-assistants coexistence model used by existing assistant support features.
- Avoid making Claude Code guidance an independent source of truth for workflow behavior.
- Validate both new-project setup and existing-workspace refresh flows.
- Use the requirement specification as the source evidence for feature scope and acceptance.

## Future Evolution Suggestions

- Add automated audits that compare canonical workflow commands with Claude Code command coverage.
- Track Claude Code onboarding friction to refine command naming and ignore defaults.
- Consider a centralized assistant capability matrix to reduce drift across assistant integrations.
