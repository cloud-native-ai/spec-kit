# Feature Detail: Todo Command

**Feature ID**: 025  
**Name**: Todo Command  
**Description**: Discover marked TODO blocks in text files and turn them into reviewable execution plans.  
**Status**: Planned  
**Created**: 2026-06-22  
**Last Updated**: 2026-06-23

## Overview

The Todo Command introduces `/speckit.todo` as a workflow command for harvesting explicitly marked, multiline TODO blocks from workspace text files. It converts those blocks and their source context into a reviewable todo plan so project notes can become bounded, executable work without manually searching through files.

## Latest Review

Created from the 020-speckit-todo-command requirements specification. The requested scope is a new command that delegates TODO discovery to a search script, extracts fenced `SPECKIT TODO` blocks from text files, combines each block with surrounding context, and generates a todo plan for execution.

## Key Changes

1. Add a dedicated `/speckit.todo` command concept to the Spec Kit command set.
2. Define special fenced TODO marker discovery for eligible text files.
3. Require source context to accompany every extracted TODO block.
4. Convert extracted TODO blocks into grouped, reviewable execution plans.
5. Establish safe handling for no-op, malformed, oversized, and risky TODO content.
6. (Phase 1) Data model formalised: `TodoBlock`, `MalformedBlock`, `TodoGroup`, `TodoPlan`, `ExecutionBatch` with canonical state transitions and 17 cross-entity invariants. Evidence: `data-model.md`.
7. (Phase 1) CLI contract for `search-todo.sh` locked: flags, exit codes, JSON schema, discovery rules (D-1…D-8), context rules (C-1…C-4), and FR traceability. Evidence: `contracts/search-todo-cli.md`.
8. (Phase 1) End-to-end user quickstart published, demonstrating embed → scan → review → confirm → execute → verify loop. Evidence: `quickstart.md`.

## Implementation Notes

- The command is an AI chat instruction, not a shell command users run directly.
- The discovery source is expected to be the repository-provided `search-todo.sh` script.
- The marker string is provisional and may be finalized during clarification if needed.
- Planning and execution should preserve the existing Spec Kit safety pattern of reviewable, bounded changes.

## Future Evolution Suggestions

- Add filtering by path, marker priority, or TODO category.
- Support machine-readable output for downstream automation.
- Track completed TODO blocks back to their source markers for cleanup suggestions.
