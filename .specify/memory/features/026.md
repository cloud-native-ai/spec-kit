# Feature Detail: Dependency Security

**Feature ID**: 026  
**Name**: Dependency Security  
**Description**: Automated dependency vulnerability scanning and supply chain security for the published package.  
**Status**: Draft  
**Created**: 2026-06-05  
**Last Updated**: 2026-06-05

## Overview

As a published PyPI package with runtime dependencies (`typer`, `rich`, `httpx`, `platformdirs`, `readchar`, `truststore`), the project has no automated vulnerability scanning or supply chain security measures. This feature covers enabling Dependabot or equivalent tooling and establishing a dependency update policy.

## Latest Review

Gap identified during DFX analysis (Design for Security). No `.github/dependabot.yml`, no Snyk/Safety configuration, and no SBOM generation found.

## Key Changes

1. Add `.github/dependabot.yml` for automated dependency update PRs (pip ecosystem).
2. Add a `pip-audit` or `safety` check to CI workflow for known vulnerability detection.
3. Document the dependency update policy (frequency, review expectations).
4. Consider pinning or constraining transitive dependencies for reproducible installs.

## Implementation Notes

- Dependabot supports the `pip` ecosystem and can target `pyproject.toml` directly.
- `pip-audit` is maintained by the Python Packaging Authority and integrates well with CI.
- `truststore` is only required for Python >= 3.10; vulnerability scanning should cover conditional deps.
- The `httpx[socks]` dependency pulls in `socksio`; ensure transitive deps are also scanned.

## Future Evolution Suggestions

- Generate SBOM (Software Bill of Materials) as part of the release process.
- Add license compliance scanning for dependencies.
- Consider signing releases with Sigstore for provenance verification.
- DFX reference: Design for Security (DFSec).
