# Implementation Plan: Create Skills Skill

**Branch**: `008-create-skills-skill` | **Date**: 2026-05-10 | **Spec**: [requirements.md](requirements.md)
**Input**: Specification from `.specify/specs/008-create-skills-skill/requirements.md`

## Summary

Extract the detailed Skill creation workflow currently embedded in `templates/commands/skills.md` into a dedicated `create-skills` Skill under `skills/create-skills/`. Keep `/speckit.skills` as the explicit command entrypoint, but reduce it to orchestration responsibilities: parse the user's target Skill, check whether that Skill exists, delegate missing targets to `create-skills`, and delegate existing targets to `improve-skills`. Preserve existing Skill quality expectations, registry behavior, and validation guidance through the extracted Skill.

## Technical Context

**Language/Version**: Python 3.8+ for CLI/script tests; Bash for project automation scripts; Markdown/YAML for command and Skill assets  
**Primary Dependencies**: Typer, Rich, httpx, platformdirs, readchar, mcp CLI, hatchling; no new runtime dependency required  
**Storage**: File-based repository artifacts (`templates/commands/skills.md`, `skills/create-skills/SKILL.md`, optional mirrored `.specify/skills/create-skills/SKILL.md`, `.specify/instructions.md` registry)  
**Testing**: pytest for existing script/layout/resource-ID checks; targeted prompt-asset assertions or review checks for command and Skill content  
**Target Platform**: Linux/macOS/Windows-compatible repository assets generated by the Spec Kit CLI; current workspace validation on Linux  
**Project Type**: Python CLI/template repository with Markdown prompt templates, Bash automation scripts, and reusable Skill directories  
**Performance Goals**: Maintainers can identify creation-vs-improvement routing in under 2 minutes; reviewers can locate detailed creation guidance in `create-skills` without searching the command template  
**Constraints**: Preserve SDD workflow; do not add unsupported AI providers; keep command templates concise; keep Skill guidance progressively disclosed; avoid duplicating full creation logic in both command and Skill; preserve compatibility symlink model  
**Scale/Scope**: One command template refactor, one new Skill directory, supporting documentation/registry updates, and targeted regression tests for existing Skill installation behavior
